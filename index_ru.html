Введение Что находится в шаблоне и для чего его можно использовать

Благодарим Вас за приобретение шаблона веб-сайта. Данное руководство покажет, как работать с шаблоном, как его редактировать и сделать "живым" сайтом.

Контент всех страниц написан на HTML, а динамическая анимация - на JavaScript.

Разархивирование шаблона

В распакованном архиве шаблона находятся 3 папки: "documentation", "screenshots" и "site". Также вы увидите архив "sources_############.zip", который нужно распаковать.

Папка sources содержит исходные файлы темы. Она защищена паролем и ее можно открыть только при помощи программ WinZip (Windows OS) и StuffitExpander (MAC OS).

Оба приложения можно скачать бесплатно для ознакомительного использования:

  • WinZip
  • StuffitExpander

При разархивировании файла "sources_############.zip" нужно будет ввести пароль, который можно найти на странице скачивания шаблона (откройте ссылку в электронном письме от нашей компании).

После распаковывания шаблон будет состоять из 4 папок: "documentation", "screenshots", "site" и "sources".

Общие сведения

Данное руководство предназначено для ознакомления с особенностями работы фреймворка, с его возможностями и принципами. Поясняется структура шаблона, основанного на фреймворке FullJS v.3, также описан принцип построения и создания страниц контента, особенности инициализации собственных скриптов. Фреймворк предназначен для создания анимированных шаблонов с возможностью тонкой настройки, легкого быстрого редактирования. От предыдущих версий FullJS-фреймворков версия 3 отличается универсальностью, что позволяет создавать разнообразные типы шаблонов.

Во фреймворк FullJS v.3 встроена возможность построения мобильной версии сайта в автоматическом режиме. Также сайты являются адаптивными, то есть сохраняют нормальный вид и перестраиваться при узких разрешениях или недостаточной величине экрана. Фреймворк имеет набор параметров и модулей, которые можно настраивать.

Компоненты фреймворка

Фреймворк включает в себя следующие библиотеки и технологии:

  1. jQuery - библиотека JavaScript, фокусирующаяся на взаимодействии JavaScript и HTML. Библиотека jQuery помогает легко получать доступ к любому элементу DOM, обращаться к атрибутам и содержимому элементов DOM, а также манипулировать ними. Также библиотека jQuery предоставляет удобный API для работы с AJAX.
    jquery.com
  2. Twitter Bootstrap v.3.x.x - набор инструментов для создания адаптивных сайтов и веб-приложений. Включает в себя HTML и CSS шаблоны оформления для типографики, веб-форм, кнопок, меток, блоков навигации и прочих компонентов веб-интерфейсов, включая JavaScript расширения.
    getbootstrap.com
  3. Ajax - подход к построению интерактивных пользовательских интерфейсов веб-приложений, заключающийся в "фоновом" обмене данными браузера с веб-сервером. В результате, при обновлении данных веб-страница не перезагружается полностью, и веб-приложения становятся быстрее и удобнее.
  4. HTML5 History API - библиотека для работы с историей адресной строки браузера.
    https://github.com/browserstate/history.js/
  5. Font Awesome - векторный шрифт с огромным количеством иконок, стиль которых легко изменять просто через CSS. Поддерживается Retina дисплеями.
    http://fortawesome.github.io/Font-Awesome/
  6. LESS - это динамический язык стилей, который обеспечивает следующие расширения CSS: переменные, вложенные блоки, миксины, операторы и функции.
    http://lesscss.org/

Особенности фреймворка

Фреймворк имеет следующие свойства и особенности:

  • модульная структура ядра: ядро фреймворка состоит из модулей, которые могут активироваться, тем самым изменяя и раскрывая функционал ядра.
  • упор на CSS3 анимацию и креативность: продукт FullJS v.3 призван заменить Flash, поэтому особый упор в нем сделан на анимацию, которая является отличительной особенностью данных шаблонов. И соответственно на анимацию посредством CSS3, а не с помощью скриптов. Скриптовая анимация, хотя уже и стала стандартом, является менее быстродейственной, чем анимация CSS3.
  • использование Bootstrap: использование Bootstrap в качестве основы для сайта делает код более понятным, стандартизированным и универсальным.
  • адаптивность: в эпоху такого стремительного развития технологий неадаптивные сайты уже перестают восприниматься как полнофункциональные, и адаптивность входит в стандарт, поэтому данный фреймворк было решено сделать именно адаптивным.
  • мобильная версия : для ограниченных ресурсов мобильных устройств было решено сделать свою мобильную версию, которая автоматически создается и настраивается разработчиком. Это экономит мобильные ресурсы и делает сайт доступным на большем количестве устройств.
  • гибкая система настройки сайта под разные устройства: в данном типе сайтов существует механизм, который позволяет управлять правилами для элементов, их отображением и поведением через CSS или Javascript.
  • загрузка страниц через Ajax: позволяет не перезагружать сайт полностью, а загружать отдельные страницы, что экономит время и ресурсы.
  • History API: использование данной библиотеки избавляет от решеток («хешей») в адресной строке для современных браузеров, и страницы имеют обычный адрес (в Internet Explorer 8, 9 все же используется хеш #); добавляет функциональности (так как работают кнопки браузера Back и Forward).
  • пополняющаяся база разнообразных плагинов под фреймворк: база плагинов постоянно растет, дополняется новыми галереями и свойствами.
  • поиск по сайту: в сайт встроен механизм поиска, и можно без труда найти нужную страницу по какому-либо слову или фразе.
  • сохранение всех положительных свойств предыдущих версий продукта FullJS: были убраны ненужные свойства предыдущих версий FullJS и создан новый фреймворк, который сохраняет все плюсы данного продукта.

Cтруктура папок и файлов

Файловая структура обязательных для работы фреймворка файлов и папок выглядит следующим образом:

  • ajax – папка с подгружаемыми страницами через Ajax *.html, *.php
    • .htaccess - файл конфигурации веб-сервера для папки Ajax (см. описание .htaccess в корневой папке)
    • 404.html - страница, которая отображается при генерации браузером 404-й ошибки (запрашиваемая страница не найдена). Обычно эта страница является подгружаемой (через Ajax), но может и находится непосредственно в файле index.html
    • search.php - файл, отвечающий за поиск текста в страницах сайта
  • assets - папка с набором разнообразных полезных примеров, скриптов, стилей от Twitter Bootstrap
  • css – папка с файлами каскадных таблиц стилей
    • bootstrap.css - базовые стили фреймворка Twitter Bootstrap v3.x.x
    • style.css - главный файл с набором стилей для сайта
  • fonts - папка с набором иконочных шрифтов Font Awesome
  • img - папка с изображениями для сайта
  • js - папка для скриптов шаблона
    • jquery.js - фреймворк jQuery
    • history.js - библиотека для работы HTML5 History API
    • device.js - библиотека для определения типа и операционной системы устройства, на котором запускается сайт
    • core.js - файл-ядро фреймворка FullJS v.3
    • bootstrap.min.js - фреймворк Twitter Bootstrap v.3.x.x
    • jquery.cookie.js - jQuery плагин для работы с куки (cookies) браузера
    • jquery.mousewheel.min.js - плагин для отслеживания событий колесика мыши
    • script.js - главный файл с необходимыми скриптами для шаблона
  • less - набор less файлов
  • index.html - файл с базовым html деревом
  • .htaccess - файл дополнительной конфигурации веб-сервера Apache, а также подобных ему серверов. Позволяет задавать большое количество дополнительных параметров и разрешений для работы веб-сервера в отдельных каталогах (папках), таких как управляемый доступ к каталогам, переназначение типов файлов и т.д., без изменения главного конфигурационного файла

Данная структура является типичной для FullJS v.3 шаблонов, однако, как правило, шаблоны дополняются скриптами, файлами стилей, подгружаемыми страницами, изображениями и т. д. Но по умолчанию такая структура папок все равно должна сохраняться.

Редактирование шаблона

Подготовка к работе

Зайдите в корень папки сайта. Каждый из редактируемых файлов находится в папке с соответствующим типом. Редактирование шаблона проходит путем правки файлов *.html, *.php, *.css (либо *.less) и *.js. Для правки файлов, вам понадобятся некоторые знания этих языков: HTML, CSS (или LESS), PHP, Javascript, а также редактор. Все данные типы файлов откроет и самый простой текстовый редактор, например, Notepad (для Windows), однако, редактировать им неудобно. Поэтому, лучше скачайте один из следующих редакторов:

  • SublimeText http://www.sublimetext.com/
  • Dreamviewer https://creative.adobe.com/products/dreamweaver
  • Notepad++ http://notepad-plus-plus.org/

Для просмотра сайта локально понадобится локальный сервер, например, Wamp (http://www.wampserver.com/en/). Это необходимо для работы Ajax запросов и php-файлов. Если вы используете Mozilla Firefox последних версий, возможно, ваш сайт будет полностью функционировать даже без использования локального сервера (http://www.mozilla.org/en-US/firefox/new/).

Вы можете воспользоваться следующими туториалами для настройки локального сервера:

Порядок строения структуры HTML

В теге head подключаются как базовые файлы, необходимые для работы фреймворка так и пользовательские файлы *.css и *.js (а также мета-данные). Подключать файлы нужно в определенном порядке. Вначале должны идти файлы со стилями:

<link rel="stylesheet" href="css/style.css">

Далее после файлов-стилей подключаются файлы со скриптами:

<script type="text/javascript" src="js/script.js"></script>

Специфика HTML страниц фреймворка FullJS v.3

Сайт под управлением фреймворка FullJS v.3 имеет немного специфическую, но довольно стандартную и простую для понимания структуру. В данном примере указано HTML код для сайта, в котором страницы расположены в файле index.html:

<article id="content" data-follow="location" data-type="switcher" data-behavior="scroll">
    <div data-id="page1"></div>
    <div data-id="page2"></div>
    <div data-id="page3"></div>
    <div data-id="page4"></div>
</article>

Существует набор атрибутов, с которыми взаимодействует ядро фреймворка. Их предназначение описано ниже:

  • data-follow="location" - атрибут, который отвечает за отслеживание переключения адресной строки и включает реакцию браузера на это изменение.
  • data-type="switcher" - атрибут, который указывает предназначение блока c контент-страницами. Для элемента HTML дерева, в котором находятся страницы для переключения, этот атрибут должен быть равен значению "switcher".
  • data-behavior="scroll" - атрибут, отвечающий за поведение скрипта для переключения страниц. Например, если целью является создание одностраничного сайта, страницы которого будут располагаться друг за другом и будет осуществляться реакция на прокрутку страницы с переключением страниц в адресной строке браузера, то данному атрибуту следует присвоить значение "scroll". Если данное поведение не требуется, то значению атрибута следует присвоить пустую строку data-behavior="".

Как изменить прелоадер сайта?

Часто для более простой последующей кастомизации используется скрипт spin.min.js. Он позволяет создавать прелоадер с настраиваемыми параметрами. Его подключение и инициализация находятся в файле script.js. О подкючении пользовательских скриптов читайте в соответствующем разделе FAQ.

Пример настройки прелоадера:

function spinnerInit(){    
  var opts = {
    lines: 11,
    length: 10,
    width: 5,
    radius: 14, 
    corners: 1,
    color: '#fff',
    speed: 1.3,
    trail: 5
  },
  spinner = new Spinner(opts).spin($('#webSiteLoader')[0]);
}
$(document).ready(function(e){
  spinnerInit();
});
$(window).load(function(){
  setTimeout(function(){
    $("#webSiteLoader").fadeOut(500, function(){
      $("#webSiteLoader").remove();                
    });
  }, 1500);
});

В данном примере прелоадер сайта инициализируется на событие ready документа и скрывается на событие load окна.
Также прелоадер может быть представлен Gif-анимированным изображением. В этом случае в скрипте spin.min.js пропадает необходимость, его можно и нужно отключить, удалив скрипт инициализации и js файл.

Формат подстраниц

Фреймворк позволяет размещать страницы либо в index.html, либо в одельных файлах, которые будут находиться в указанной папке и динамически загружаться (по умолчанию это папка Ajax). Если страница находится внутри index.html, то, как правило, это обычный DIV-элемент разметки, который находится в блоке c атрибутами data-follow="location" data-type="switcher" (атрибут data-behavior может отсутствовать):

<article id="content" data-follow="location" data-type="switcher" data-behavior="">
…
</article>

Страница внутри вышеописанного блока обычно имеет следующий вид:

<div data-id="readmore">
<div class="container">
  <div class="row">
    <div class="col-lg-12">
      <h2>Read More</h2>
      <p>Aenean sapien massa, ultricies in risus at, consectetur.</p>
    </div>
  </div>
</div>
</div>

Внешняя страница, которая будет загружаться посредством Ajax и находиться в одноименной папке - это обычная страница, созданная по нормам HTML5 (в данном примере с использованием классов Bootstrap):

<!doctype html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>Read More</title>
</head>
<body>
  <div class="container">
    <div class="row">
      <div class="col-lg-12">
        <h2>Read More</h2>
        <p>Aenean sapien massa, ultricies in risus at, consectetur.</p>
      </div>
    </div>
  </div>
</body>
</html>

Файлы скриптов и файлы стилей на загружаемых страницах прописывать не рекомендуется. В этих случаях лучше подключить скрипт в script.js.

<article id="content" data-follow="location" data-type="switcher" data-behavior="scroll">
    …
    <div data-id="page4"></div>
    <div data-id="page5">
      <!-- your content here -->
    </div>
</article>

Структура меню шаблона

Главное меню в шаблоне FullJS v.3 имеет некоторую специфику. Чтобы запустить связь меню со страницами сайта, в меню должны быть заданы некоторые атрибуты, которые одновременно являются параметрами для работы фреймворка:

  • data-follow="location" - атрибут, который отвечает за отслеживание переключения адресной строки и включает реакцию браузера на это изменение.
  • data-type="navigation" - атрибут, который обозначает тип для данного элемента. Для меню указано, что типом и соответственно предназначением элемента является навигация по сайту.
<nav data-follow="location" data-type="navigation">
    <ul>
        <li><a href="./page1">page1</a></li>
        <li><a href="./page2">page2</a></li>
        <li><a href="./page3">page3</a></li>
        <li><a href="./page4">page4</a></li>
    </ul>
</nav>

Ссылки

Ссылки в шаблоне могут быть как внешние (которые ведут на другие домены), так и внутренние (ссылки на внутренние страницы документа, то есть те страницы, которые находятся в индексном файле, либо, которые были загружены через Ajax). Внутренние ссылки нужно начинать с "./" Например: 

<a href="./about"></a>
В этом случае фреймворк будет знать, что нужно показать внутреннюю страницу контента.

Ajax страницы

Фреймворк позволяет также использовать подгрузку страниц контента с помощью технологии Ajax. Дополнительные страницы находятся в папке Ajax. Создаем файл (somePage.html) с контентом и помещаем в эту папку. Далее для загрузки такой внешней страницы на любую ссылку шаблона в атрибут href пишем имя, созданного ранее *.html файла, с расширением *.html:

<a href="./somePage.html  "></a>

Hа любой контейнер со страницами добавляем атрибут data-flags="ajax":

<article id="content" data-follow="location" data-type="switcher" data-flags="ajax">
    <div data-id="page1"></div>
    <div data-id="page2"></div>
    <div data-id="page3"></div>
    <div data-id="page4"></div>
</article>
Страница, загруженная через Ajax, будет подгружена в этот контейнер и будет отображаться так же, как и любая другая внутренняя страница шаблона. Данный атрибут указывает на то, что страница является подгружаемой, и запускает соответствующие алгоритмы для загрузки страницы.

Структура главного файла стилей

Редактирование файла стилей осуществляется посредством использования одного из редакторов перечисленных выше. Структура главного css-файла style.css, как правило, следующая:

  • подключение других css-файлов через import
  • описание стилей для body, заголовков, параграфов и прочих основных элементов
  • описание стилей для header-а
  • описание стилей для контента
  • описание стилей для footer-а
  • описание дополнительных стилей
  • корректирование стилей через media-queries для придания сайту адаптивности (@media (min-width: 1200px) {…})
  • корректирование стилей для версии сайта под планшеты (блок html.tablet)
  • корректирование стилей для мобильной версии сайта (блок html.mobile)

Подключение CSS-файлов

Для того, чтобы не перегружать индексный файл, рекомендуется подключать файлы стилей в главном файле стилей сайта style.css. Подключение таким образом не является жестким требованием, но упрощает структуру сайта.

Пример:

@import 'photoswipe.css';

Cтили для разных лейаутов

Как правило, практически все стили для шаблона находятся в файле style.css. При открытии сайта на портативных устройствах движок сайта определяет, является ли устройство планшетом или мобильным телефоном. При этом на тег html навешиваются соответствующие классы. Ниже перечислены классы, которые добавляются к тегу html при открытии на разных устройствах (десктопном компьютере, планшетном ПК, мобильном телефоне):

html.desktop{}
html.tablet{}
html.mobile{}

Для того, чтобы стили применялись для определенного типа устройств, их следует начинать с указанием принадлежности к соответствующему типу лейаута: 

html.tablet h1 {
  font-size: 45px;
}

По умолчанию все стили относятся к десктопной версии сайта, то есть прописывать html.desktop перед стилями необязательно. Они и так будут относиться к десктопной версии по умолчанию.

Адаптивные стили

Для управления отображением сайта на разных разрешениях применяются media-queries. Все стили, которые будут применяться для определенного разрешения, следует указывать внутри блока media (max-width: …px){…}.
Например:

@media (max-width: 1024px) {
  h1{
    font-size: 45px;  
  }
}
@media (max-width: 650px) {
  h1{
    font-size: 35px;  
  }
}

Подключение скриптов

Рекомендуется подключать скрипты не обычным способом в index.html в теге "head", а в файле script.js. Этот файл является главным файлом настройки и инициализации ядра и плагинов. Подключение скриптов можно производить через вызов функции includе, указав аргументом для нее скрипт для подключения:

include('js/superfish.js');

if (!FJSCore.mobile){
  include('js/superfish.js');

То есть данный скрипт будет загружаться только при открытии на десктопном компьютере и на планшете.

$(document).ready(function(e){

либо на событие окончания загрузки для окна:

$(window).load(function(e){
…

Мобильная версия сайта

Общие сведения

Во фреймворк FullJS v.3 встроена возможность построения мобильной версии сайта в автоматическом режиме. Данная версия будет отображаться при условии, что шаблон загружен на мобильном телефоне. Это нужно для того, чтобы на планшетах сайт отображался в полноценном desktop-варианте с адаптивным лейаутом, так как современные варианты данных портативных устройств оборудованы достаточно мощной аппаратной частью и практически не уступают стационарным ПК. Мобильная версия предназначена для отображения шаблона сайта на портативных устройствах с небольшими размерами экрана, например, мобильный телефон или иные устройства, которые снабжены интернет браузером.

Структура

Дерево документа мобильной версии имеет следующую структуру:

<div id="mobile-wrap">
    <div id="mobile-header">
        <h1></h1>
        <select id="mobile-navigation">
        …
        </select>
    </div>
    <div id="mobile-content">
        <div data-id="page1"></div>
        <div data-id="page2"></div>
        …
        <div data-id="pageN"></div>
    </div>
    <div id="mobile-footer"></div>
</div>

Фреймворк при условии, что переменная FJSCore.mobile равна истине, выстраивает в документе данную структуру автоматически, при этом удаляя дерево документа для desktop-версии. В блок 

<div id="mobile-content"></div>
переносятся все страницы desktop версии без изменений, имена страниц сохраняются. На основе меню сайта с указанным атрибутом data-type="navigation" выстраивается мобильная версия меню в виде select. Также добавляется выпадающее меню, подуровни подменю не ограничены. Это меню находится в хедере сайта, туда же и переносится тег h1, который можно использовать как блок для хранения логотипа. В блок <div id="mobile-footer"></div> переносится блок с классом .copyright (ссылка privacy policy) и .follow-links (список соц. ссылок).

Редактирование

Если вам нужно, чтобы тот или иной блок не был скопирован в мобильную версию, на этот блок достаточно добавить класс .desktop-only. Для отображения же только на мобильной версии существует другой класс - .mobile-only.
Мобильная версия создается автоматически по умолчанию, но есть возможность отключить эту функцию. Как уже описывалось выше, скрипты для мобильной версии нужно заключать в условие: 

 if(FJSCore.mobile){}.

Поскольку дерево мобильной версии создается с помощью ядра фреймворка, то для редактирования этого дерева используются *.js-скрипты. Для примера обернем все элементы хедера и футера в элемент div с классом .container:

$('#mobile-header>*').wrapAll('<div class="container"></div>');
$('#mobile-footer>*').wrapAll('<div class="container"></div>');

Отключение мобильной версии

Отключение мобильной версии происходит путем инициализации свойства mobileVersion для объекта FJSCore mobileVersion. В Core.js необходимо задать mobileVersion: false. Подробнее о настройке ядра смотрите в пункте FAQ по настройке Сore.js.

Core.js. Параметры и настройка

В данном блоке описана работа главного скрипта фреймворка core.js. Описаны основные переменные, их предназначение, события и примеры кода для использования.

Параметры ядра

В core.js существует определенный набор параметров, использование которых раскроет потенциал ядра, а понимание этих параметров упростит создание сайтов:

  • defState: '' - переменная, отвечающая за ссылку для страницы по умолчанию (главной). Обычно это пустая строка, и ссылка в адресной строке будет выглядеть следующим образом: http://domenname/site/ ;
  • defMobileState:'' - переменная, отвечающая за ссылку для главной страницы для мобильной версии;
  • emptyNavigationText:'-- navigate to --' - переменная, отвечающая за текст, отображаемый в select-меню для мобильных устройств. Переход на пункт меню с данным текстом будет осуществляться, если требуемый пункт меню не найден. Например, в случае со страницей PrivacyPolicy;
  • ajaxFolder: "/ajax/" - переменная, указывающая название папки со страницами подгружаемыми через Ajax;
  • indexFile: 'index.html' - переменная, которая указывает главный индексный файл. В нем должны быть подключены все необходимые *.css и *.js файлы;
  • mobileVersion: true - переменная ядра, отвечающая за автоматическое создание и запуск мобильной версии;
  • mobileFolowLinks: true - переменная ядра, отвечающая за запуск механизма перестроения ссылок для соцсетей в мобильной версии сайта. Иногда, при ограниченном месте для отображения данных ссылок, такой механизм будет полезен;
  • state - переменная, которая показывает название текущей страницы. Это то самое название, которое пишется в адресной строке;
  • prevState - переменная, которая показывает название предыдущей страницы. Иногда нужна для манипуляций с анимацией страниц;
  • mobile - переменная ядра, которая показывает, что сайт загружен на мобильном устройстве;
  • tablet - переменная ядра, которая показывает, что сайт загружен на планшете.

Менять значение переменных пока можно только в core.js. Впоследствии будет реализован механизм для инициализации всех изменяемых параметров ядра в других скриптах.

Сейчас получить значение переменной можно через обращение к главному объекту FJSCore. Например, выражение FJSCore.mobile вернет значение true/false, которое обозначает запущен ли сайт на мобильном телефоне.

События ядра

Ядро core.js работает, используя события. Самым важным и часто используемым является событие changeLocation. Данное событие срабатывает каждый раз при изменении url сайта. Это и переходы по меню, и просто переходы по ссылкам, и загрузка сайта. Контроль над данным событием позволяет произвести необходимые действия при переходе от страницы к странице. Например, настроить анимацию, поставить какие-то условия. Данное событие применяется к документу.
Этот пример выводит в консоль событие и новое значение url:

$(document).on('changeLocation', function(e,d){
  console.log(e);
  console.log(d);
});

Дальше можно расширить скрипт, например добавлением условия с проверкой на мобильное устройство:

$(document).on('changeLocation', function(e,d){
  if (FJSCore.mobile) {
    …
  } else {
    …
  }
});

Модуль responsiveContainer

Данный блок предназначен для ответа на вопросы: "Что такое модуль responsiveContainer?", "Зачем он нужен?" и "Как его настроить?".

ResponsiveContainer - модуль ядра core.js для перестроения сайта на разных разрешениях. Так как для анимации необходимо, чтобы у блоков было абсолютное позиционирование, это лишает сайт реакции на изменение размеров окна. Эта проблема решается подключением модуля ядра responsiveContainer. Важно, чтобы страница была создана, придерживаясь правил и структуры для создания адаптивных сайтов (в данном случае Bootstrap). Вызов этого модуля производится в блоке окончания загрузки окна и имеет следующий вид:

$(window).load(function(){
  FJSCore.modules.responsiveContainer({
    elementsSelector: '#other_pages>div',
    defStates: ''
  });
});

Модуль responsiveContainer может работать в 2-х режимах: изменять min-height у элемента body в зависимости от высоты контента и добавлять скролл на страницу. Это достигается изменением свойств при инициализации модуля.
Параметры плагина responsiveContainer (указаны значения по умолчанию):

  • container: '' - селектор контейнера для страниц, может быть как родительским элементом страницы, так и самой страницей;
  • elementsSelector: '.content-pages>div' - селектор страниц для последующей манипуляции с ними;
  • type: 'outer' - режим работы плагина. При значение outer скролл будет появляться у боди, при inner - у элементов DOM-дерева, которые указаны в параметре elementsSelector;
  • affectSelectors: 'header, footer' - селекторы элементов, которые влияют на высоту страницы. Это могут быть, например, футер и хедер, либо какие-то панели. При отсутствии хедера или футера в документе их высота просто не будет учитываться, даже если они заданы параметром;
  • deltaHeight: 0 - значение, на которое можно изменить значение высоты при каких-либо действиях. Например, при переходе на определенную страницу происходит выдвижение какого-то элемента сверху сайта, и при этом нужно увеличить высоту сайта, хотя этот селектор не указан в affectSelectors, потому что не влияет на высоту сайта на всех страницах, а только на определенной. В этом случае следует вызвать событие плагина updateDeltaHeight для body, указав желаемое значение в виде строки: 

    $('body').trigger('updateDeltaHeight', '300');

    При этом следует помнить, что необходимо обнулить величину deltaHeight при переходе на другую страницу, где больше нет необходимости в приросте высоты:

    $('body').trigger('updateDeltaHeight', '0');
  • activePageSelector: '.activeSubPage' - селектор класса активной страницы. Именно по этому селектору выбирается та страница, по которой нужно сделать вытягивание сайта;
  • resizeDisableSelector: '.disableResize' - иногда необходимо на страницах определенного вида не производить учет высоты. Для этого существует этот параметр. В такой странице необходимо на элемент DOM-дерева добавить класс disableResize, либо какой-то другой, передав его как значение для данного свойства. На таких страницах пересчет высоты учитываться не будет;
  • defStates: '' - когда значение страницы равно одному из значений, указанных в этом свойстве, то пересчет высоты производиться не будет и будет равен свойству css min-height, указанному в style.css. Если необходимо перечислить страницы, на которых не требуется пересчитывать высоту, нужно переопределить данный параметр, как указано ниже. Обычно такие действия требуются для сплеш-страниц либо страниц с галереями. Следует обратить внимание, что первым значением в перечислении данных параметров указана пустая строка, затем идут перечисления адресов галерей. Это значения, которые идут после последнего слеша в адресной строке браузера: 

    FJSCore.modules.responsiveContainer({
  defStates: ',fashion,nature,food,animals'
});

Пример настройки модуля responsiveContainer

Рассмотрим случай, если в сайте нет футера и хедера (или только футера / хедера), понятно, что и учитывать высоту неоткуда. И другой случай: если они есть, но учитывать их высоту по каким-либо причинам не нужно. Тогда можно переназначить параметр affectSelectors таким образом (в случае, если нужно учитывать только высоту хедера): 

FJSCore.modules.responsiveContainer({
  affectSelectors: 'header'
});

Если футер и/или хедер позиционированы абсолютно, то для работы плагина можно добавить свойство margin (поле) для CSS-селектора страниц (значения для elementsSelector) в style.css, где прописаны основные стили сайта. Верхнее и нижнее значения свойства margin будут учитываться при расчете высоты сайта.

В style.css: 

.content-pages>div{
  margin: 200px 0;  
}
header, footer{
  position: absolute;
}

В script.js:

FJSCore.modules.responsiveContainer({
  elementsSelector: '.content-pages>div',
  affectSelectors: ''
});

Существуют следующие классы/типы по умолчанию:

  • name – Имя пользователя. Проверка: цифры не допускаются, не менее 2 символов;
  • email – Email пользователя;
  • state – Страна (Область). Проверка: цифры не допускаются, не менее 2 символов;
  • phone – Телефон. Проверка: только цифры, не менее 5 символов. Дополнительные допустимые неалфавитные знаки: "space", "+" and "-";
  • fax – Факс. Проверка: только цифры, не менее 5 символов. Дополнительные допустимые неалфавитные знаки: "space", "+" and "-";
  • file – Загрузка файла. Проверка: максимальный размер файла для загрузки 10 Мегабайт. Можно загружать только файлы следующего формата: doc, docx, txt, pdf, zip, rar.

  • @Required – Этот атрибут данных используется как указатель опциональных полей при заполнении формы.

Также, вышеупомянутый тег label может включать элементы span с классами error-message и empty-message, которые содержат сообщение об ошибке, если информация введена неверно или поле пустое.

Описание

Структура HTML внутри формы гибкая, единственное условие - поля ввода должны быть расположены внутри тега label, с соответствующим его предназначению классом.

По умолчанию, в ней находятся следующие классы/типы:

  • name – Имя пользователя. Проверка: цифры не допускаются, не менее 2 символов;
  • email – Email пользователя;

Также, вышеупомянутый тег label может включать элементы span с классом error, который представляет собой сообщение об ошибке, если данные были введены неверно.

$(window).load(function(){

Для инициализации используйте следующие параметры:

В случае успешной подписки, владелец сайта и посетитель получат два разных письма, подтверждающих подписку. Почтовый адрес владельца сайта подгружается из файла sForm.js, а адрес посетителя берется из поля e-mail формы подписки.

Политика использования Cookies

Cookies - это небольшие текстовые файлы, которые веб-сайты создают на компьютерах пользователей. Они используются для идентификации пользователей и улучшения качества использования веб-сайта. Cookies являются важной частью многих функций веб-сайтов.

Например, cookies позволяют онлайн магазинам запоминать, какие товары вы добавили в корзину. Также они сохраняют данные входа на сайт или предоставляют ценную статистику использования и информацию владельцам сайтов.

Если у вас есть веб-сайт, он почти наверняка использует cookies. Их главная функция - изучать количество и интересы пользователей через специальные инструменты, например Google Analytics. Также они могут использоваться для отображения объявлений, или – если у вас есть онлайн-магазин – быть ключевой частью системы электронной коммерции.
